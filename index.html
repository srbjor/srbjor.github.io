180103通路串扰相关论文阅读小结

Posted on 2019-09-22

全局路径串扰网络 A global pathway crosstalk network

概述：

鉴于生物系统的复杂性，生物的生理反应常常是通过协调发挥作用的，因此理解通路直接的相互作用和串扰，对于理解细胞和更复杂的系统显得极其重要。此文构建了一个哺乳动物的全局通路串扰网络，并且此网络遵循幂律分布：

p(k)~k^(-γ),γ=1.45

一般说来，从大数据量的基因列表中提取潜在的生物学信息的一种方法是识别这些列表中显著改变的通路。而分析通路又分为两种方法，一种是从统计学意义上使用超几何分布估算，另一种是采用了非参数方法来确定微阵列实验中的通路，来确定是否发生了显著改变。

除了基因表达谱以外，蛋白质-蛋白质相互作用也提供了基因如何执行器生物学功能的有用信息。通过建立蛋白质的全局相互作用网络，可以解释细胞存货和生长所必需的的整体组织和相互作用。

已经使用过类似方法的文章：

Genome-wide protein–protein interaction screenings based on yeast two-hybrid technique have been performed in budding yeast (Uetz et al., 2000), fruit fly (Giot et al., 2003), worm (Li et al., 2004), human (Stelzl et al., 2005) and other organisms. On the other hand, literature-based curation and data-mining processes have also generated substantial amounts of protein interaction data (Rajagopalan and Agarwal, 2005; Ramani et al., 2005). Network-based methods have been used to analyze these interaction data and gain insights into the mechanism by which biological systems operate (Barabasi and Oltvai, 2004; Xia et al., 2004).

此文采取了一种计算方法，通过系统地结合通路数据和交互网络数据来探索通路相互作用。该方法基于这样的假设：如果在这两条通路之间检测到显着更多的蛋白质相互作用，高于随机发生的相互作用或相互影响，即认为该通路间发生了串扰。此文展示如何建立一个基于串扰的通路网络，并基于基因表达谱数据将其应用于进一步了解结直肠癌的发展和转移。

分析方法：

这篇文章的方法比较古老，第一步是搜集通路数据和蛋白质相互作用数据，基本上都是从网络上下载，做了一些基本的预处理：如删掉所含基因过少的通路。

2.3 路径串扰网络的构建

去除含有少于 5 个基因或超过 100 个基因的通路来产生一组串扰分析通路。基因过多的通路可能过于一般，基因太少的通路可能没有足够的生物学内容。这些大小的临界值是任意设置的。
通过执行 Fisher Exact 测试评估任何给定的一对通路之间的基因重叠。
计算任何两个通路之间的蛋白质相互作用的数量。对于每个通路对，首先去除两条通路共有的基因，然后计算这两条通路之间的所有蛋白质相互作用。
估计每个通路对的蛋白质相互作用计数的背景分布。每条通路如下随机化：浏览给定路径中的所有基因。如果一个基因没有任何相互作用，就跳过它。如果一个基因有相互作用，首先计算它与之相互作用的基因的数量，然后从与相同或相似数目的基因相互作用的蛋白质相互作用数据集中随机抽取一个基因，并用这个新选择的基因取代原始通路基因。一旦两条通路被随机化，则执行步骤 3 以计算它们之间的相互作用的次数。该随机化步骤重复 1000 次。
使用包含以下数字的 2×2 列联表进行所有通路对的单侧 Fisher 精确检验：n，N - n，r，R - r 其中 n 表示原始通路之间的相互作用计数，N 表示 r 表示所有通路对的总相互作用计数，r 表示经过 1000 轮随机化后相应随机化通路对之间的相互作用计数的平均值，R 表示所有随机化通路对在 1000 轮随机化之后的总相互作用计数的平均值。零假设是两条通路与所有相互作用之间真实相互作用的比率（n /N）与随机相互作用与所有随机相互作用的比率（r /R）相同。在我们的分析中，我们只关注 n / N 显着高于 r / R 的通路对。Fisher 精确检验使用 FDR BH 程序（1995 年的算法）调整 P 值，以说明多重假设检验。我们还使用第二种方法来评估两种通路之间相互作用计数的重要性。具体而言，我们将真正的相互作用的数量与在相同通路对上进行的 1000 个排列的随机相互作用计数进行排序。经验 P 通过计算随机交互计数高于或等于真实交互计数的置换数目，然后将该数目除以置换总数计算值。Fisher 精确检验 P-值和经验 P-值非常好地相关（数据未显示）。
将调整后的 Fisher 精确检验 P 值 P<0.05 的所有通路对拉到一起以构建网络，其中节点是通路，边表示两个通路之间的串扰。在下面的修剪步骤中，删除了在某些路径中由于基因冗余而导致的两种“冗余”边缘，以清理网络。
(a)在步骤 2 中鉴定的具有显着基因重叠的两条通路之间的所有边缘被认为是不具有信息性的，因此从网络中移除。请注意，我们的目的是发现不同生物活动之间的串扰。两个通路在基因成员方面彼此显着重叠的通路对代表相似的生物学，因此被排除在网络之外。
（b）两条重叠的通路可能都与相同的通路相互作用。在这种情况下，两条边被认为是多余的，其中一条被删除。例如，通路 A 具有四个邻居：边 B，C，D 和 E。如果 B 中至少 75％的基因也在 C 中，并且 B 具有比 C 更少的基因数量，则 A 和 B 之间的边缘被去除。重复这个过程，直到路径 A 和它的邻居之间不再有更多的边缘可以被删除。这个步骤对于从 GO 获得的通路是特别必要的，其中所有的儿童基因组都是父母的完整子集。如果小孩和父母的 GO 通路都与通路 A 相互作用，则在修剪过程中， A 和子 GO 通路之间的边缘将被移除。

2.4 网络聚类

通路串扰网络 PCN 中的通路基于其最短路径分布进行聚类。简言之，任何给定的一对通路之间的最短路径（SP）是使用标准的基于图形的程序来计算的。两条路径之间的距离（d）被定义为它们之间的 SP 的长度。使用简单的转化（1 / d）来产生在 0 和 1 范围内的所有通路对的相似性分数.1 / d 之间的路径和本身设置为 1。然后使用进行分层凝聚平均链接聚类。聚类结果在沿对角线对称的热图中查看（自我关联）。较深的颜色表示两个路径在 SP 长度方面接近。

事务：事务是关系型数据库的核心特性。Go中事务（Tx）是一个持有数据库连接的对象，它允许用户在同一个连接上执行上面提到的各类操作。

事务基本操作：

通过db.Begin()来开启一个事务，Begin方法会返回一个事务对象Tx。在结果变量Tx上调用Commit()或者Rollback()方法会提交或回滚变更，并关闭事务。在底层，Tx会从连接池中获得一个连接并在事务过程中保持对它的独占。事务对象Tx上的方法与数据库对象sql.DB的方法一一对应，例如Query,Exec等。事务对象也可以准备(prepare)查询，由事务创建的准备语句会显式绑定到创建它的事务。

事务注意事项

  • 使用事务对象时，不应再执行事务相关的SQL语句，例如BEGIN,COMMIT等。这可能产生一些副作用：
  • Tx对象一直保持打开状态，从而占用了连接。
  • 数据库状态不再与Go中相关变量的状态保持同步。
  • 事务提前终止会导致一些本应属于事务内的查询语句不再属于事务的一部分，这些被排除的语句有可能会由别的数据库连接而非原有的事务专属连接执行。
  • 当处于事务内部时，应当使用Tx对象的方法而非DB的方法，DB对象并不是事务的一部分，直接调用数据库对象的方法时，所执行的查询并不属于事务的一部分，有可能由其他连接执行。

Tx的其他应用场景

如果需要修改连接的状态，也需要用到Tx对象，即使用户并不需要事务。例如：

  • 创建仅连接可见的临时表

  • 设置变量，例如SET @var := somevalue

  • 修改连接选项，例如字符集，超时设置。

  • 在Tx上执行的方法都保证同一个底层连接执行，这使得对连接状态的修改对后续操作起效。这是Go中实现这种功能的标准方式。

在事务中准备语句

调用Tx.Prepare会创建一个与事务绑定的准备语句。在事务中使用准备语句，有一个特殊问题需要关注：一定要在事务结束前关闭准备语句。

在事务中使用defer stmt.Close()是相当危险的。因为当事务结束后，它会释放自己持有的数据库连接，但事务创建的未关闭Stmt仍然保留着对事务连接的引用。在事务结束后执行stmt.Close()，如果原来释放的连接已经被其他查询获取并使用，就会产生竞争，极有可能破坏连接的状态。

数据库注意事项：

  • 尽量避免空值（用0值代替空值，数据库添加NOT NULL约束）
  • 打开和关闭数据库（sql.DB）可能会导致资源枯竭；
  • 结果集没有读取完毕，或者调用rows.Close()失败，结果集会一直占用池里的连接；
  • 使用Query()执行一些不返回结果集的语句，返回的未读取结果集会一直占用池里的连接；
  • 不了解准备语句（Prepared Statement）的工作原理会产生许多额外的数据库访问。
  • Go底层使用int64来表示整型，使用uint64时应当极其小心。使用超出int64表示范围的整数作为参数，会产生一个溢出错误

标准数据库不提供批量操作

一次查询执行多条SQL语句没有明确定义，具体实现驱动器实现，会产生不可预料的结果避免这种情况。

因为事务保证在它上面执行的查询都由同一个连接来执行，因此事务中的语句必需按顺序一条一条执行。对于返回结果集的查询，结果集必须Close()之后才能进行下一次查询。用户如果尝试在前一条语句的结果还没读完前就执行新的查询，连接就会失去同步。这意味着事务中返回结果集的语句都会占用一次单独的网络往返。

WGCNA

基本概念

WGCNA其译为加权基因共表达网络分析。该分析方法旨在寻找协同表达的基因模块(module)，并探索基因网络与关注的表型之间的关联关系，以及网络中的核心基因。
适用于复杂的数据模式，推荐5组(或者15个样品)以上的数据。一般可应用的研究方向有：不同器官或组织类型发育调控、同一组织不同发育调控、非生物胁迫不同时间点应答、病原菌侵染后不同时间点应答。

基本原理

方法上来讲，WGCNA分为表达量聚类分析和表型关联两部分，主要包括基因之间相关系数计算、基因模块的确定、共表达网络、模块与性状关联四个步骤。

第一步计算任意两个基因之间的相关系数（Person Coefficient）。为了衡量两个基因是否具有相似表达模式，一般需要设置阈值来筛选，高于阈值的则认为是相似的。但是这样如果将阈值设为0.8，那么很难说明0.8和0.79两个是有显著差别的。因此，WGCNA分析时采用相关系数加权值，即对基因相关系数取N次幂，使得网络中的基因之间的连接服从无尺度网络分布(scale-freenetworks)，这种算法更具生物学意义。

第二步通过基因之间的相关系数构建分层聚类树，聚类树的不同分支代表不同的基因模块，不同颜色代表不同的模块。基于基因的加权相关系数，将基因按照表达模式进行分类，将模式相似的基因归为一个模块。这样就可以将几万个基因通过基因表达模式被分成了几十个模块，是一个提取归纳信息的过程。
